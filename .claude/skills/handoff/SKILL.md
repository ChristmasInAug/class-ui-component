# handoff — 세션 인계

작업을 중단하거나 다음 세션(다른 사람 또는 미래의 나)에게 넘길 때, 현재 상태를 프로젝트 루트의 `.handoff.md` 한 파일에 스냅샷으로 남긴다. 목적은 "내가 뭘 했나"의 일지가 아니라 **"다음 사람이 뭘 알아야 이어갈 수 있나"**를 최소한의 스캔 시간으로 전달하는 것이다.

## 언제 실행하나

- 사용자가 "핸드오프", "인계", "여기까지 정리", "세션 넘겨줘", `/handoff` 등을 요청할 때
- 컨텍스트 압축·긴 작업 전환 직전, 사용자가 상태 저장을 명시적으로 요청한 경우

## 절차

### 1. 컨텍스트를 실제로 수집한다 (추측 금지)

아래 명령을 직접 실행해서 결과를 확인한 뒤 문서를 작성한다. 실행하지 않고 기억이나 짐작으로 채우지 않는다.

```bash
date '+%Y-%m-%d %H:%M'
git branch --show-current
git status --short         # untracked 포함 변경 요약
git diff --stat            # tracked 변경 요약
```

여기에 더해 **이번 세션에서 실제로 실행한** 빌드/테스트/실행 명령과 그 출력(성공/실패, 통과한 테스트 수 등)을 근거로 사용한다.

### 2. 기존 `.handoff.md`가 있는지 확인한다

- 있으면 **덮어쓰지 않고 갱신**한다: 이번 세션에서 끝난 항목은 ⏳ → ✅로 옮기고, 여전히 유효한 미완료 항목은 그대로 carry-forward한다. 더 이상 유효하지 않은 항목(코드가 이미 바뀌어 무의미해진 것 등)은 제거한다.
- 없으면 아래 템플릿으로 새로 만든다.

### 3. 아래 형식으로 `.handoff.md`를 작성/갱신한다

```markdown
# 🤝 Handoff — <한 줄 현재 상태 요약>

> 갱신: <YYYY-MM-DD HH:MM> · 브랜치: `<branch>` · 변경: `<git status --short 요약(신규/untracked 포함) — tracked 변경은 git diff --stat>`

## ✅ 완료 (검증된 것만)
- [x] <한 일> — 증거: `<실행한 명령 / 파일:라인 / 테스트 결과>`

## ❌ 시도 → 실패 / 막힘 (반복 금지)
- [ ] <시도한 접근> — 원인: <왜 안 됐나> → 다음 대안: <있으면>

## ⏳ 다음 할 일 / 미착수
- [ ] <다음 단계> — 시작점: `<파일 / 명령>`

## 📌 컨텍스트 메모
- <이어받는 사람이 모르면 헤맬 결정·제약·함정·환경 설정. 없으면 이 섹션 생략>
```

각 섹션이 비어 있으면 `- 없음`으로 명시한다(섹션 자체를 생략하지 않는다. 단 컨텍스트 메모는 정말 특기할 게 없으면 섹션째 생략 가능).

## 핵심 원칙

- **✅는 "검증된 것만".** 각 항목에 증거(실행한 명령·파일:라인·테스트 결과)를 반드시 붙인다. 증거가 없으면 ✅가 아니라 ⏳로 내린다. "되겠지"로 완료 표시하지 않는다.
- **❌는 "시도했지만 안 된 접근 + 원인".** 다음 세션이 같은 막다른 길을 반복하지 않게 하는 것이 목적이다.
- **⏳는 "바로 이어갈 다음 단계 + 시작점(파일/명령)".** 미착수 항목도 빠짐없이 적는다.
- **인수자 관점으로 쓴다.** "내가 뭘 했나"가 아니라 "다음 사람이 뭘 알아야 이어가나"를 기준으로 고른다.
- **간결·스캔 가능하게.** 각 항목 한 줄. 산문 단락 금지.
- **기존 파일은 갱신, 덮어쓰기 금지.** 끝난 항목은 ✅로 옮기고 유효한 미완료는 carry-forward.

## 완료 후

- `.handoff.md` 경로와 변경 요약(신규 작성 vs 갱신, 추가/이동된 항목 수)을 한두 문장으로 보고한다.
- `.handoff.md`는 세션 간 작업 메모이며 게시 대상이 아니다. 이 저장소는 GitHub Pages로 공개 배포되므로(`docs/resource/TROUBLE_SHOOTING.md` 참고) `.handoff.md`가 `.gitignore`에 포함되어 있는지 확인하고, 없다면 추가할지 사용자에게 확인한다.
